# AGENTS.md

## 開発時のルール
 - **日本語で返信をすること**
 - コードはコードスタイルに従うこと
    - **特にワンライナーifやfor文は使用禁止です**
 - ユーザーの求める課題に対して最小限の変更かつ最もシンプルな実装を行うこと
    - これは著書「ルールズオブプログラミング」や「ロバストPython」でも述べられ重視されている内容である
        - **最小限の変更** : なるべく既存の概念/モジュール/コード/実装を使用するような変更
        - **最もシンプルな実装** : コードレビュー等を行ったときにストレートに読むことが出来るシンプルなコード
 - 1作業1変更に必ずユニットテストとE2Eテストを実装し、完了前に「`pnpm run lint`」、「`pnpm run test`」、「`pnpm run e2e`」を走らせ、パス後コミットを行うこと
 - ブランチ名は以下のルールに従うこと
    - **機能追加** : feature/{機能名}
    - **機能以外の追加(役職等)** : feat/{機能名}
    - **リファクタリング** : refactor/{リファクタ名}
 - コミット名は以下のルールに従うこと
    - **機能追加** : feat: {コミット詳細}
    - **リファクタリング** : refactor: {コミット詳細}
    - **修正** : fix: {コミット詳細}
    - **既存処理変更** : change: {コミット詳細}
 - PR発行時、**タイトルと本文も必ず日本語で記載**
 - 開発時に得た知識は「.gemini/skills/」フォルダ直下にそれぞれフォルダを作り、mdファイルで知識を入れる
    - README.mdも更新すること

## コードスタイル
- コメントは**必ず日本語で記述する**こと
- ワンライナーifは書かず必ず{}を付けて書くこと
- インデントはなるべく小さく、アーリーリターンを使用すること
- TypeScript
    - ワンライナーifは書かず必ず{}を付けて書くこと
    - [GoogleのTypeScript Styleガイド](https://google.github.io/styleguide/tsguide.html)に従うこと
    - オブジェクトを定義したい場合は基本的に`Interface`を使い、ユニオン型等で特殊な型を定義したい場合にtypeを使うべき
    - ユニットテストは「`tests`」フォルダに記載(vitestを使用)
    - E2Eテストは「`e2e`」フォルダに記載(playwrightを使用)
- React
    - 「`import React from 'react';`」は不要
    - TailWind CSSでGUIのスタイリングを行うこと
    - 最新のReactではReact.FC等が非推奨になったため直接関数宣言を行い、引数の型を書く関数スタイルで記述つすること
    - 引数の型は「関数名」 + Propsで行うこと
    - export等は関数宣言時に実施
    - 1ファイル1コンポーネントとし、コンポーネントの粒度に応じてフォルダ分けを行う
    - ディレクトリ構造は以下とする
        - src: ソースディレクトリ
            - components: 再利用可能なコンポーネントフォルダ、ここのコンポーネントは基本的にステートレス
                - extra: 外部のモジュールから取ってきたコンポーネント
                - blocks: partsを組み合わせたコンポーネント、blocksを組み合わせたものもここに置く
                - parts: UIの最小構成のコンポーネント、これ以上分解することが出来ない
            - logics: 生tsファイルを置く場所
            - slices: スライスした`zustand状態`(`StateCreator<T>`のエクスポートした変数)置き場、ここのStoreをインポートしてはいけない
            - feature: アプリケーションウィンドウに配置されたコンポーネント置く場所、ビジネスロジック等がここに含まれる再利用不可
            - App.tsx: アプリケーションウィンドウコンポーネント
            - useStore.ts: `zustand`の状態管理置き場
            - type.ts: 他のファイルで使用する。型置き場
        - test: テストコードを置く場所
    - Reactの`useMemo`等のパフォーマンス向上トリックは使用しない(React Compilerで自動的に追加される)
        - もし使用しなければならない場合はRust等の利用を検討する
    - コンポーネントはなるべくステートレスになるように設計しfeatureディレクトリ内の肥大化を防ぐ
    - バックエンドからのデータ取得は`Suspose`を使うこと、`useEffect`は基本使用しない
    - 状態管理は`zustand`を使用し、`useState`は使用しない
        - 必ずグローバルの`useStore`からデータを取得すること
    - 記載例:
    ```ts

    interface GreetingProps {
        name: string;
    };

    function Greeting({ name }: GreetingProps) {

        const func = () => {
            console.log('hello')
        }

        return <h1>Hello {name}</h1>
    }

    ```
